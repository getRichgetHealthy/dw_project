# 监管数据集市项目计划

## 1. 项目概述

### 1.1 项目背景
建设统一的监管数据集市，满足金融监管机构的数据报送要求，实现监管数据的标准化、集中化管理。

### 1.2 项目目标
- 整合全行业务数据，满足监管报送要求
- 建立统一的监管数据标准和口径
- 实现监管报表自动化生成
- 支持实时监控和预警

## 2. 业务需求

### 2.1 监管报送范围
| 模块 | 主要内容 | 报送频率 |
|------|----------|----------|
| 反洗钱 | 大额交易报告、可疑交易报告 | 实时/日 |
| 客户尽职调查 | KYC信息、客户身份识别 | 实时 |
| 风险指标 | 资本充足率、流动性风险指标 | 月/季 |
| 业务数据 | 存贷款数据、理财业务数据 | 月/季 |
| 现场检查数据 | 监管现场检查所需数据 | 按需 |

### 2.2 核心功能
- 数据采集与整合
- 数据校验与清洗
- 异常数据监控
- 历史数据存档

## 3. 数据模型设计

### 3.1 主题域划分
- 客户主题：客户基本信息、关系人信息
- 交易主题：交易流水、账务数据
- 风险主题：风险指标、预警信息
- 机构主题：分支机构信息、组织架构
- 报送主题：报送记录、监管反馈

### 3.2 数据分层
```
ODS (操作数据层) -> STG (临时层) -> DW (数据仓库) -> DM (数据集市) -> RPT (报表层)
```

## 4. 技术架构

### 4.1 技术选型
- 数据存储：MySQL / PostgreSQL
- 数据开发：Airflow=
- 调度系统：内置调度

### 4.2 系统架构图
```
数据源 -> ETL -> 数据存储 -> 报表生成 -> 监管报送
```

#### 5.2.1 STG层开发
详细设计见 [STG层设计文档](docs/STG层设计.md)

**STG层表清单：**
| 表名 | 中文名 | 更新方式 | 主要清洗内容 |
|------|--------|----------|--------------|
| stg_customer | 客户信息清洗表 | 全量 | 姓名trim、证件标准化、性别编码、哈希脱敏 |
| stg_account | 账户信息清洗表 | 增量 | 类型编码、状态标准化、余额填充 |
| stg_transaction | 交易流水清洗表 | 增量 | 金额转换、渠道编码、大额标识、对手信息脱敏 |
| stg_contract | 合同信息清洗表 | 增量 | 状态标准化、逾期天数、担保方式编码 |
| stg_organization | 机构信息清洗表 | 全量 | 层级标准化、地区编码 |
| stg_risk_indicator | 风险指标清洗表 | 增量 | 指标标准化、阈值设置 |

**核心清洗规则：**
- 统一编码：证件类型、性别、渠道、风险等级
- 数据脱敏：证件号哈希、手机号掩码、账户号掩码
- 格式标准化：日期格式、金额单位、特殊字符处理
