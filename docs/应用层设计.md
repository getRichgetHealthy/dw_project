### 2.2 报表详情

#### RPT_AML_001 客户风险分布统计表
- 报表维度：机构、风险等级、时间
- 报表指标：各风险等级客户数量、高风险客户占比、风险等级变化趋势、大额交易笔数/金额、可疑预警次数

#### RPT_AML_002 大额交易明细表
- 报表字段：报告编号、客户姓名、证件号码、账户号、交易金额、交易时间、交易类型、交易渠道、报送状态、报送时间
- 筛选条件：交易金额>=50万

#### RPT_REG_001 资本充足率报表
- 报表指标：核心一级资本充足率>=7.5%、一级资本充足率>=8.5%、资本充足率>=10.5%
- 告警规则：低于监管要求时高亮显示

### 2.3 报表技术实现(示例SQL)
```sql
SELECT 
    org_id,
    risk_level,
    COUNT(*) AS customer_count,
    SUM(CASE WHEN large_trans_count > 0 THEN 1 ELSE 0 END) AS large_trans_customer,
    SUM(trans_amount_12m) AS total_trans_amount
FROM dm_aml_customer_risk
WHERE dt = '${bizdate}'
GROUP BY org_id, risk_level;
```

## 四、DM层ETL开发

### 4.1 反洗钱ETL流程

**客户风险评级汇总ETL逻辑：**
1. 从DW层客户表抽取客户基本信息
2. 关联风险评分表获取风险分数
3. 汇总近1/3/12月交易金额
4. 统计大额交易笔数、可疑预警次数
5. 计算最终风险等级(高>=70分, 中>=40分, 低<40分)
6. 写入dm_aml_customer_risk

### 4.2 监管指标ETL流程

**资本充足率计算ETL逻辑：**
1. 从DW层抽取资本相关数据
2. 计算核心一级资本充足率 = 核心一级资本 / 风险加权资产
3. 计算一级资本充足率 = 一级资本 / 风险加权资产
4. 计算资本充足率 = 资本总额 / 风险加权资产
5. 判断是否达标(资本充足率>=10.5%)
6. 写入dm_regulatory_capital

### 4.3 ETL调度依赖

```
dw_xxx -> dm_aml_customer_risk -> dm_aml_statistics
       -> dm_aml_large_trans_report -> RPT_AML_002
       -> dm_aml_suspicious_trans -> RPT_AML_003
       
dw_risk_indicator -> dm_regulatory_capital -> RPT_REG_001
                  -> dm_regulatory_liquidity -> RPT_REG_002
                  -> dm_regulatory_loan_quality -> RPT_REG_003
```
